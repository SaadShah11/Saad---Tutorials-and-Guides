🍺 Homebrew - The Package Manager

Homebrew is the essential package manager for macOS. It lets you install, update, and manage software from the command line with simple commands.

Step 1: Install Homebrew

Open Terminal (press ⌘ + Space, type "Terminal") and paste this command:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
⚠️ Note: You'll be prompted to enter your Mac password. The characters won't appear as you type—this is normal. Press Enter when done.

Step 2: Add Homebrew to Your PATH

After installation, Homebrew will show you commands to add it to your PATH. For Apple Silicon Macs (M1/M2/M3), run:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

For Intel Macs, run:

echo 'eval "$(/usr/local/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/usr/local/bin/brew shellenv)"

Step 3: Verify Installation

Close and reopen Terminal, then run:

brew --version

You should see something like: Homebrew 4.x.x

💡 Tip: Run brew doctor to check if your Homebrew installation is healthy and see any recommended fixes.

📦 Essential Homebrew Commands

Here are the commands you'll use most often:

Command Description
brew install <package> Install a command-line tool (e.g., brew install git)
brew install --cask <app> Install a GUI application (e.g., brew install --cask visual-studio-code)
brew uninstall <package> Remove a package
brew list Show all installed packages
brew search <term> Search for available packages
brew update Update Homebrew itself and package lists
brew upgrade Upgrade all installed packages to latest versions
brew upgrade <package> Upgrade a specific package
brew outdated List packages that have updates available
brew info <package> Show info about a package
brew cleanup Remove old versions of packages to free up space
brew doctor Check for potential problems with your installation

🔧 Maintaining Homebrew

Keep your system clean and up-to-date with these maintenance commands:

Regular Maintenance Routine

Run this weekly to keep everything updated:

# Update Homebrew and see what's outdated
brew update

# Show packages that need updating
brew outdated

# Upgrade all packages
brew upgrade

# Clean up old versions
brew cleanup

# Check for issues
brew doctor

One-Liner for Quick Maintenance

brew update && brew upgrade && brew cleanup
⚠️ Note: Some cask apps (like Chrome) update themselves. Run brew upgrade --greedy to force-update all casks regardless of auto-update settings.

📋 Export & Restore Your Setup

Homebrew can create a "Brewfile" that lists all your installed packages, making it easy to set up a new Mac.

Export Your Current Setup

# Create a Brewfile in your home directory
brew bundle dump --file=~/Brewfile

# View what was exported
cat ~/Brewfile

Restore on a New Mac

Copy your Brewfile to the new Mac, then run:

# Install everything from the Brewfile
brew bundle --file=~/Brewfile
💡 Tip: Store your Brewfile in a Git repo (like a dotfiles repo) so you always have it backed up!

❓ Troubleshooting

Command Not Found After Install

If you install something but the command isn't found:

  1. Close and reopen Terminal
  2. Or run: source ~/.zprofile
  3. Check it's installed: brew list | grep package-name

Permission Errors

If you get permission errors, run:

sudo chown -R $(whoami) $(brew --prefix)/*

Check Homebrew Health

brew doctor

This will identify and suggest fixes for common issues.

Reinstall Homebrew

If things are really broken, you can uninstall and reinstall:

# Uninstall Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/uninstall.sh)"

# Then reinstall using Step 1 above

📦 NVM - Node Version Manager

NVM (Node Version Manager) allows you to install and manage multiple versions of Node.js on your Mac. This is essential for working on different projects that may require different Node.js versions.

⚠️ Prerequisite: Make sure you have Homebrew installed first (see the Homebrew section above).

Step 1: Install NVM via Homebrew

Install NVM using Homebrew:

brew install nvm

Step 2: Create NVM Directory

Create the directory where NVM will store Node.js versions:

mkdir ~/.nvm

Step 3: Add NVM to Your Shell Profile

Add NVM configuration to your ~/.zshrc file. This ensures NVM is loaded every time you open a new terminal:

export NVM_DIR="$HOME/.nvm"
[ -s "$(brew --prefix nvm)/nvm.sh" ] && \. "$(brew --prefix nvm)/nvm.sh"
[ -s "$(brew --prefix nvm)/etc/bash_completion" ] && \. "$(brew --prefix nvm)/etc/bash_completion"

Or run this command to append it automatically to your ~/.zshrc:

echo 'export NVM_DIR="$HOME/.nvm"
[ -s "$(brew --prefix nvm)/nvm.sh" ] && \. "$(brew --prefix nvm)/nvm.sh"
[ -s "$(brew --prefix nvm)/etc/bash_completion" ] && \. "$(brew --prefix nvm)/etc/bash_completion"' >> ~/.zshrc
💡 Tip: The $(brew --prefix nvm) command automatically finds the Homebrew installation path for NVM, so you don't need to hardcode paths.

Step 4: Reload Your Shell

Close and reopen Terminal, or run:

source ~/.zshrc

Step 5: Verify Installation

Check that NVM is installed correctly:

nvm --version

You should see something like: 0.39.7 (version may vary)

🚀 Essential NVM Commands

Here are the most commonly used NVM commands:

Command Description
nvm install node Install the latest LTS version of Node.js
nvm install <version> Install a specific Node.js version (e.g., nvm install 20.10.0)
nvm install --lts Install the latest LTS (Long Term Support) version
nvm use <version> Switch to a specific Node.js version
nvm use node Switch to the latest installed version
nvm use --lts Switch to the latest LTS version
nvm list List all installed Node.js versions
nvm list-remote List all available Node.js versions to install
nvm list-remote --lts List only LTS versions available to install
nvm current Show the currently active Node.js version
nvm alias default <version> Set a default Node.js version for new terminal sessions
nvm uninstall <version> Uninstall a specific Node.js version
nvm which <version> Show the path to the Node.js executable for a version

💡 Common NVM Workflows

Installing and Using Node.js

Install the latest LTS version and use it:

# Install latest LTS
nvm install --lts

# Use it
nvm use --lts

# Set it as default
nvm alias default node

Working with Multiple Projects

Different projects may require different Node.js versions. Here's how to manage that:

# Install multiple versions
nvm install 18.20.0
nvm install 20.10.0
nvm install 22.0.0

# Switch between them as needed
nvm use 18.20.0  # For project A
nvm use 20.10.0  # For project B

Using .nvmrc Files

Create a .nvmrc file in your project root to automatically use the correct Node.js version:

# In your project directory, create .nvmrc
echo "20.10.0" > .nvmrc

# Then use it
nvm use
💡 Pro Tip: Add this to your ~/.zshrc to automatically switch Node versions when entering a directory with a .nvmrc file: 
# Auto-switch Node version when .nvmrc exists
autoload -U add-zsh-hook
load-nvmrc() {
  local node_version="$(nvm version)"
  local nvmrc_path="$(nvm_find_nvmrc)"

  if [ -n "$nvmrc_path" ]; then
    local nvmrc_node_version=$(nvm version "$(cat "${nvmrc_path}")")

    if [ "$nvmrc_node_version" = "N/A" ]; then
      nvm install
    elif [ "$nvmrc_node_version" != "$node_version" ]; then
      nvm use
    fi
  elif [ "$node_version" != "$(nvm version default)" ]; then
    echo "Reverting to nvm default version"
    nvm use default
  fi
}
add-zsh-hook chpwd load-nvmrc
load-nvmrc

🔧 NVM Troubleshooting

NVM Command Not Found

If you get nvm: command not found after installation:

  1. Make sure you've created the ~/.nvm directory: mkdir ~/.nvm
  2. Verify you've added the NVM configuration to your ~/.zshrc (see Step 3)
  3. Check that the paths use $(brew --prefix nvm) instead of hardcoded paths
  4. Reload your shell: source ~/.zshrc
  5. Close and reopen Terminal
  6. Verify NVM is installed via Homebrew: brew list nvm

Node Version Not Persisting

If your Node.js version resets after closing Terminal:

# Set a default version
nvm alias default 20.10.0

# Verify it's set
nvm list

Permission Errors

If you encounter permission errors, make sure NVM directory has correct permissions:

sudo chown -R $(whoami) ~/.nvm

Check NVM Installation

Verify NVM is properly installed and configured:

# Check if NVM is installed via Homebrew
brew list nvm

# Check NVM version
nvm --version

# Check NVM directory
echo $NVM_DIR

# Check Homebrew NVM path
brew --prefix nvm

# List installed Node versions
nvm list
